# زوّد (Zawwid) — تطبيق توصيل بقالة سريع (Q-Commerce)

تطبيق كامل مستوحى من فكرة تطبيقات التوصيل السريع (مثل نينجا، Getir، إلخ) — **بكود وتصميم وهوية أصلية بالكامل**. المشروع يتكوّن من 3 أجزاء:

```
zawwid/
├── backend/    → API (Node.js + Express + SQLite)
├── mobile/     → تطبيق الموبايل للعملاء + المدير (Expo / React Native)
└── README.md   → هذا الملف
```

## ⚠️ ملاحظة قانونية
هذا مشروع مبني من الصفر بكود واسم وتصميم مختلف تمامًا عن أي تطبيق موجود. الاسم "زوّد" والألوان والشعار كلها أصلية، وتقدر تغيّرها بحرية. لو تبي تنشر التطبيق تجاريًا، تأكد إنك ما تستخدم شعارات أو أسماء أو صور من تطبيقات أخرى.

## 1) تشغيل الباك-إند (API)

```bash
cd backend
npm install
npm run seed     # يجهز قاعدة بيانات SQLite ببيانات تجريبية
npm run dev       # يشغل السيرفر على http://localhost:4000
```

**حسابات تجريبية بعد الـ seed:**
| الدور | رقم الجوال | كلمة المرور |
|---|---|---|
| مدير (admin) | 0500000000 | admin123 |
| عميل تجريبي | 0511111111 | 123456 |

**فحص سريع:** افتح `http://localhost:4000/api/health` لازم يرجع `{"status":"ok"}`.

### أهم نقاط الـ API
- `POST /api/auth/signup` / `POST /api/auth/login` — تسجيل ودخول
- `GET /api/categories` — الأقسام
- `GET /api/products?category=&q=` — المنتجات (فلترة وبحث)
- `GET/POST/PUT/DELETE /api/cart` — السلة
- `POST /api/orders` — إتمام الطلب (checkout)
- `GET /api/orders` — طلبات المستخدم
- `GET /api/orders/:id` — تفاصيل وتتبع طلب معيّن
- `GET /api/orders/admin/all` + `PUT /api/orders/:id/status` — للمدير: إدارة الطلبات
- `GET /api/admin/stats` — إحصائيات لوحة التحكم
- كل مسارات `POST/PUT/DELETE` على `/api/products` و `/api/categories` تتطلب دخول بحساب `admin` أو `store_manager`

## 2) تشغيل تطبيق الموبايل (Expo)

```bash
cd mobile
npm install
npx expo start
```

بعدها:
- امسح رمز QR بتطبيق **Expo Go** (نزّله من App Store / Google Play) لتجربته على جوالك الحقيقي
- أو اضغط `a` لتشغيله على محاكي أندرويد، أو `i` لمحاكي آيفون (يحتاج Xcode)

### 🔴 مهم: ربط التطبيق بالسيرفر
افتراضيًا التطبيق يتصل بـ `http://localhost:4000/api`. هذا يشتغل تمام إذا شغّلت Expo Web. لكن:
- **على جوال حقيقي عبر Expo Go**: لازم تستخدم عنوان IP جهاز الكمبيوتر بدل localhost (مثال: `http://192.168.1.5:4000/api`) لأن الجوال ما يقدر يوصل لـ localhost بتاع جهازك.
- **على محاكي أندرويد**: استخدم `http://10.0.2.2:4000/api`

غيّر القيمة في `mobile/app.json` تحت `expo.extra.apiUrl`.

## 3) الميزات المُنفذة

### للعميل
- تصفح الأقسام والمنتجات + بحث
- تفاصيل المنتج واختيار الكمية
- السلة (إضافة/تعديل/حذف)
- إتمام الطلب (دفع عند الاستلام أو بطاقة)
- تسجيل دخول / إنشاء حساب
- سجل الطلبات + **تتبع لحظي لحالة الطلب** (تحديث تلقائي كل 10 ثواني)

### للمتجر/الإدارة (نفس التطبيق، لحساب admin أو store_manager)
- لوحة تحكم بإحصائيات (طلبات نشطة، إجمالي مبيعات، عملاء، منتجات على وشك النفاد)
- إدارة الطلبات: فلترة حسب الحالة + تقديم الحالة خطوة بخطوة أو إلغاء
- إدارة المنتجات: إضافة / تعديل / حذف (يشمل السعر، الوحدة، المخزون، القسم)

---

## 4) الخطوات التالية المقترحة (لو تبي تطوّرها أكثر)
- رفع الصور الفعلية للمنتجات (حاليًا `image_url` نص فاضي، أضف رفع ملفات عبر `multer` الموجود في الباك-إند)
- بوابة دفع حقيقية (مثل Moyasar أو Tap للسوق السعودي) بدل "بطاقة" الوهمية
- تتبع موقع السائق على خريطة حية (يحتاج تكامل مع خدمة خرائط + WebSocket)
- إشعارات Push عند تغيّر حالة الطلب (Expo Notifications)
- نشر الباك-إند على سيرفر حقيقي (Render, Railway, VPS) بدل التشغيل المحلي
- Build نسخة APK/IPA فعلية عبر `eas build` (يتطلب حساب Expo)
